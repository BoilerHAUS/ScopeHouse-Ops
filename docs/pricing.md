# ScopeHouse Pricing Hypotheses v1

## Purpose

Document the first pricing ranges ScopeHouse should test.

These are not final prices.
They are working hypotheses for:

- customer interviews
- design partner follow-up
- founder-led service offers
- paid pilot conversations
- early landing page framing

## Pricing principles

1. Price against user pain and time saved, not feature count.
2. Keep the first pricing model simple enough to explain in one minute.
3. Use service revenue to learn before forcing self-serve SaaS behavior.
4. Do not underprice high-friction founder work.
5. Separate homeowner buying logic from pro buying logic.

## Core pricing structure

Start with three lanes:

1. homeowner per-project pricing
2. pro monthly subscription pricing
3. founder-led service pricing

## 1. Homeowner pricing hypothesis

### Proposed model

Per-project pricing.

### Why

Homeowners usually think in terms of one renovation, not ongoing software seats.

They care about:

- getting the plan organized
- making cleaner decisions
- comparing quotes
- reducing mistakes before and during the job

A per-project model fits how they buy.

### Initial range to test

- low test: $79 per project
- middle test: $129 per project
- upper test: $149 per project

### What this should include

- one active project
- guided intake
- scope builder
- budget and schedule basics
- decision log
- exportable project summary
- AI-assisted drafting and summarization where relevant

### Best use case for this offer

Homeowners planning medium to large renovations where project confusion has visible cost.

### Risk

If the project is too small, even this price will feel unnecessary.

### Working hypothesis

Homeowners will accept a per-project price more easily than a subscription if the value is tied to one serious renovation and the outputs feel tangible.

## 2. Solo pro pricing hypothesis

### Proposed model

Monthly subscription.

### Why

Solo GCs, consultants, and design-build operators think in terms of recurring business use, repeated client workflows, and ongoing project control.

A monthly price matches:

- ongoing client work
- repeated project setup
- repeat use across jobs
- the need for a stable internal system

### Initial range to test

- low test: $99 per month
- middle test: $129 per month
- upper test: $149 per month

### What this should include

- multiple active projects
- cleaner project control across client jobs
- quote comparison support
- decision and change tracking
- exportable client-facing summaries
- structured project record across scope, budget, schedule, docs, and changes

### Best use case for this offer

Small renovation pros who want stronger control without heavyweight contractor software.

### Risk

If ScopeHouse is seen as too light, pros will compare it against spreadsheets and notes.
If it is framed too broadly, they will compare it against full contractor suites and reject it on missing back-office features.

### Working hypothesis

Solo pros will pay a monthly price if ScopeHouse clearly improves client project control and saves them from scattered records, weak quote comparison, and missing decision history.

## 3. Small team pricing hypothesis

### Proposed model

Monthly subscription for small firms.

### Why

Small firms need:

- more than one active project
- repeated process use
- shared project visibility
- cleaner client communication

But this should still stay below full contractor ERP framing.

### Initial range to test

- low test: $249 per month
- middle test: $299 per month
- upper test: $399 per month

### What this should include

- team use across multiple active projects
- stronger export and client-facing workflow value
- more repeatable process structure
- more durable project records across jobs

### Best use case for this offer

Small firms or design-build teams that need more discipline but do not want heavy enterprise software.

### Risk

This segment may demand features before product fit is proven.

### Working hypothesis

Small teams will only convert if ScopeHouse already feels credible in repeated use and if one partner or operator clearly owns the process.

## 4. Founder-led service pricing hypothesis

### Proposed model

One-time service engagements tied to live renovation planning or project cleanup.

### Why

This is the fastest path to early revenue and grounded learning.

Services help:

- fund product learning
- surface real project artifacts
- improve onboarding insight
- create case-study material
- test willingness to pay before scaled SaaS behavior exists

### Initial range to test

#### Offer A, light planning support

- $300 to $500

Use for:

- guided project setup
- intake cleanup
- draft scope organization
- light summary output

#### Offer B, quote comparison and project organization

- $600 to $900

Use for:

- organizing scope against vendor quotes
- identifying gaps, overlaps, and missing items
- building a cleaner project record

#### Offer C, deeper founder-led setup

- $1000 to $1500

Use for:

- larger renovation setup
- more involved scope and control cleanup
- repeated sessions
- higher complexity or more stakeholders

### Best use case for this offer

Users with real pain now who want help immediately and are open to founder-led work.

### Risk

If service work expands too far, it can consume product-building time.

### Working hypothesis

Services will close sooner than software in early validation because users already pay for partial help, planning support, consultant input, or the hidden cost of project confusion.

## Packaging guidance

### Homeowner offer

Frame as:

- one renovation
- one project record
- one clear planning and control layer

### Pro offer

Frame as:

- ongoing control for client projects
- repeated workflow value
- cleaner project communication and records

### Service offer

Frame as:

- founder-led planning and project cleanup
- faster clarity for a live project
- practical help, not consulting theater

## What not to do yet

Do not:

- publish too many tiers
- create seat-based complexity early
- add enterprise pricing language
- force annual contracts
- bundle heavy support into low monthly pricing
- underprice founder time

## Suggested pricing tests

### Test 1, interview test

Use language like:

For a serious renovation, would this need to feel like a one-time planning purchase or an ongoing tool?

For a small pro, would this feel more natural per project or monthly?

### Test 2, service-first test

Offer founder-led setup at the lower end first and observe:

- response rate
- close rate
- objection patterns
- actual time required

### Test 3, landing page framing test

Test these phrases:

- per-project planning and control for serious renovations
- monthly project control for small renovation pros
- founder-led setup and quote comparison support

### Test 4, conversion threshold test

Track where real resistance appears:

- under $100
- around $129
- around $149
- above $299 monthly
- above $900 service work

## Recommendation

Use this starting structure:

- homeowner, $129 per project anchor
- solo pro, $129 per month anchor
- small team, $299 per month anchor
- founder-led service, $600 to $900 as the main middle offer

This gives you:

- a simple story
- clear segment separation
- room to move down or up after live feedback

## What success looks like

This pricing work is useful if it helps answer:

- which segment converts first
- whether the first money comes from software or service
- whether per-project or monthly framing is more natural for the buyer
- where resistance comes from, price or workflow trust
