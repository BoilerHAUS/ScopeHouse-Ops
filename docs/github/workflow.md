# GitHub Workflow

Use docs to define work before creating issues or opening pull requests.

## Workflow

1. Update the relevant source-of-truth doc in `docs/`.
2. Place the work in the right milestone in [`milestones.md`](./milestones.md).
3. Map executable issue candidates in [`issue-map.md`](./issue-map.md).
4. Open focused issues with a clear problem, goal, scope, acceptance criteria, and constraints.
5. Open small pull requests that tie back to one issue or one doc change at a time.

## Rules

- Prefer one outcome per issue.
- Keep milestones outcome-based, not activity-based.
- Update docs in the same change when the source of truth moves.
- Use the PR guide for final change summaries and verification notes.
