# Repo Structure

## Top level

- `README.md`: repo overview and entrypoint
- `docs/`: documentation hub for current and historical operating docs

## Docs structure

- `product/`: founder direction, MVP scope, and product-facing business decisions
- `engineering/`: repo conventions and technical documentation structure
- `github/`: workflow, milestones, issue mapping, and PR guidance
- `operations/`: operating cadence, launch prep, and business operations docs
- `adr/`: architecture decision records for repo and documentation choices
- `archive/`: inactive or historical material kept for reference
