# Engineering Rules

This repo is documentation-first. Keep structure predictable and lightweight so docs stay usable as active operating assets.

## Rules

1. Keep docs in markdown.
2. Prefer one topic per file.
3. Use relative links inside `docs/`.
4. Update [`docs/README.md`](../README.md) when docs move or new active docs are added.
5. Record structural decisions in [`docs/adr/`](../adr/README.md).
6. Move stale or replaced material to [`docs/archive/`](../archive/README.md).
7. Create GitHub issues from repeated pain, not one-off comments.
8. Keep source-of-truth files current; delete duplication instead of letting it drift.
