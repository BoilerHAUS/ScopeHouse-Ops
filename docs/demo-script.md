# ScopeHouse Demo Script v1

## Purpose

Create a short, repeatable live demo script for ScopeHouse.

This demo should show the core planning and control loop clearly.

It is not a feature walkthrough.
It is a story about turning a messy renovation into a cleaner project record.

## Demo length

Target:

8 to 12 minutes

## Core message

ScopeHouse helps bring renovation work under control by creating one clear record for scope, budget, schedule, documents, decisions, and changes.

## Demo setup

Use a realistic but simple sample project.

Best sample:

- medium-size kitchen and main floor renovation
- homeowner with active quote collection
- a few key decisions already in motion
- at least one quote comparison example

Why this works:

- relatable for homeowners
- still credible for pros
- enough complexity to show real value

## Demo flow

### 1. Frame the problem

Say:

Most renovation projects do not break because people do not care. They break because the record is scattered. Scope, quotes, decisions, and changes end up spread across notes, texts, spreadsheets, and memory.

Then say:

What ScopeHouse does is bring that project into one working system before the confusion gets more expensive.

Value moment:

The buyer understands this is about control, not generic task management.

### 2. Create the project

Show:

- new project creation
- basic project details
- project type or stage

Say:

This is where the project starts becoming a real record instead of a pile of loose information.

Value moment:

The project gets a home.

### 3. Run intake

Show:

- goals
- rooms or phases
- constraints
- timing
- budget range

Say:

Instead of collecting information randomly, ScopeHouse pulls the key planning details into a structured intake.

Value moment:

Messy ideas start becoming structured input.

### 4. Build the scope draft

Show:

- draft scope generated from intake
- editable scope structure
- how rough project intent becomes something clearer

Say:

This is where a rough project starts becoming something you can compare quotes against and make decisions around.

Value moment:

The user sees the jump from idea to working scope.

### 5. Bring budget and schedule into view

Show:

- budget frame or budget basics
- schedule phases or milestone view

Say:

Most people do not need perfect forecasting on day one. They need a cleaner frame for money and timing so the project stops floating.

Value moment:

The project starts to feel grounded.

### 6. Show decision tracking

Show:

- one or two example decisions
- who decided what
- when it changed

Say:

This matters because renovation confusion often comes from decisions getting lost, revisited, or remembered differently later.

Value moment:

The buyer sees the control layer, not just planning.

### 7. Show quote comparison or change tracking

Choose one based on the audience.

#### For homeowners

Show quote comparison.

Say:

This is where ScopeHouse helps compare quotes against a real project record instead of trying to judge them from memory.

Value moment:

The buyer sees money and scope connected.

#### For pros

Show decision or change tracking.

Say:

This is where ScopeHouse helps keep client projects cleaner as details move and decisions evolve.

Value moment:

The pro sees project discipline without heavyweight software.

### 8. Show export or summary

Show:

- project summary
- exportable output
- clean handoff view

Say:

This is important because the value is not trapped inside the app. You leave with a cleaner project record you can review, share, and act on.

Value moment:

The output feels tangible.

### 9. Close the story

Say:

The point of ScopeHouse is simple. It helps turn renovation chaos into a structured project record before and during execution, so scope, quotes, decisions, and changes stay under control.

## Audience adjustments

### Homeowner version

Lean harder on:

- quote comparison
- family decision clarity
- avoiding expensive confusion
- one place for the whole project

### Pro version

Lean harder on:

- cleaner client project control
- repeated workflow value
- decision history
- structured records without contractor ERP overhead

## Demo mistakes to avoid

Do not:

- start with AI
- click every feature
- explain backend logic
- spend too long in navigation
- show empty states for too long
- over-explain edge cases
- sound like generic project management software

## Demo checklist

Before the demo, confirm:

- sample project is populated
- one quote example exists
- one decision example exists
- one export or summary is ready
- the story matches the audience

## Ending options

### If the lead is a design partner candidate

If this lines up with the kind of project control problem you are dealing with, the next step would be a deeper workflow session.

### If the lead is a service candidate

If the fastest path is to get your live project organized, we can also do that with you through a founder-led setup.

### If the lead is a pro prospect

If this feels relevant across your client projects, the next step would be to look at how it fits your current workflow.

## Reminder

The buyer should leave the demo remembering three things:

- one clear project record
- less quote and decision confusion
- more control before mistakes compound
